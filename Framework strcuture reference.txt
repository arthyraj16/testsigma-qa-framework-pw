1. PROJECT OVERVIEW
================================================================================

This is a professional-grade Playwright test automation framework for both
Web UI and REST API testing. It follows SDET best practices with:
- Page Object Model (POM) for Web tests
- Client abstraction pattern for API tests
- GitHub Actions CI/CD pipeline with Allure reporting
- Parallel test execution
- Secret management via environment variables
- TypeScript for type safety

4. API TEST SUITE
================================================================================

LOCATION: tests/api/

TECHNOLOGY STACK:
- Playwright APIRequestContext
- TypeScript
- Client abstraction pattern
- Bearer token authentication

TEST STATISTICS:
✓ 21 tests implemented and passing
✓ 5 test endpoints covered (Applications, Environments, Projects, Uploads, Execution Results)
✓ 100% coverage on GET operations
✓ CRUD operations for Environments
✓ Parallel test execution support

API CLIENT ARCHITECTURE:

BaseApiClient (base-api-client.ts)
├── Request/Response handling
├── Header management
├── Timeout configuration
├── Retry logic
└── Error handling

Specialized Clients:
├── ApplicationsClient     → GET /applications, GET /applications/{id}
├── EnvironmentsClient     → GET/POST/PUT/DELETE /environments (CRUD)
├── ElementsClient         → GET /elements operations
├── ProjectsClient         → GET /projects operations
├── UploadsClient          → File upload operations
├── ExecutionResultsClient → GET/POST /execution_results, polling
└── TestDataProfilesClient → Test data profile operations

API TEST SPECS:

Applications Tests:
- List applications (200)
- Get individual application
- Verify response schema
- Authentication validation (401)

Environments Tests:
- List environments
- Create new environment
- Get environment details
- Update environment
- Delete environment
- Parallel environment creation
- Schema validation
- Authentication validation (401)
- Error handling (404 non-existent)

Projects Tests:
- List projects
- Authentication checks
- Non-existent resource handling

Uploads Tests:
- File upload operations
- Upload list retrieval

Execution Results Tests:
- Trigger execution
- Get execution status
- Poll until completion
- Error handling

ENVIRONMENT VARIABLES REQUIRED:
- TESTSIGMA_API_KEY (or TESTSIGMA_CICD_API_KEY): Bearer token for API authentication

RESPONSE HANDLING:
All API responses wrapped in standardized ApiResponse<T> interface:
{
  status: number,
  statusText: string,
  ok: boolean,
  data: T | null,
  headers: Record<string, string>,
  response: APIResponse,
  error?: string
}

6. CI/CD PIPELINE IMPLEMENTATION
================================================================================

FILE: .github/workflows/playwright-ci.yml

WORKFLOW TRIGGERS:
✓ Push to main branch
✓ Pull requests targeting main branch
✓ Manual trigger (workflow_dispatch)
✓ Optional: Scheduled nightly runs (commented out, can enable)

PARALLEL JOBS ARCHITECTURE:

Job: web-tests (Matrix)
├── Strategy: Run across 3 browsers in parallel
│   ├── Chromium
│   ├── Firefox
│   └── WebKit
├── Steps:
│   ├── Checkout code
│   ├── Setup Node.js 18
│   ├── npm ci (reproducible install)
│   ├── Install Playwright browsers
│   ├── Run tests: npx playwright test --project={browser}
│   └── Upload Allure results (always, even on failure)
└── Env: CI=true, TESTSIGMA_API_KEY from secrets

Job: api-tests
├── Steps:
│   ├── Checkout code
│   ├── Setup Node.js 18
│   ├── npm ci
│   ├── Install Playwright browsers
│   ├── Run tests: npx playwright test --project=api
│   └── Upload Allure results (always)
└── Env: CI=true, TESTSIGMA_API_KEY from secrets

Job: report (Merge & Publish)
├── Dependencies: Needs both web-tests and api-tests
├── Runs: Always (if: always())
├── Steps:
│   ├── Download all Allure result artifacts
│   ├── Merge results from web + api into single report
│   ├── Generate combined HTML Allure report
│   ├── Upload allure-report artifact (30-day retention)
│   ├── Generate job summary with pass/fail stats
│   └── Post summary to GitHub Actions UI
└── Output: Downloadable combined Allure report

PERFORMANCE OPTIMIZATIONS:
✓ Node dependencies cached via setup-node
✓ Playwright browser cache supported
✓ Parallel job execution (web + api run simultaneously)
✓ Serial workers on CI for stability

ARTIFACT MANAGEMENT:
✓ Allure results (raw): 5-day retention
✓ Combined report: 30-day retention
✓ Downloadable from workflow run artifacts
✓ GitHub Actions job summary visible in PR checks

SECRETS MANAGEMENT:
✓ TESTSIGMA_CICD_API_KEY stored in GitHub repo secrets
✓ Never logged to console
✓ Injected as environment variable at runtime
✓ Setup location: Settings → Secrets and variables → Actions

VISIBILITY:
✓ Clear pass/fail counts per test suite
✓ HTML report download link in artifacts
✓ Job summary shows status overview
✓ Trace files (.zip) for failed tests in artifacts
