## Code Quality Requirements
- Follow Page Object Model: page classes under /pages; specs use only page object methods
- Prefer getByRole/getByLabel locators; avoid brittle CSS/XPath
- Web-first assertions only (await expect(...).toBeVisible()); no hard waits or sleeps
- One assertion focus per test; tests must be fully independent
- Credentials only via environment variables (process.env.*)
- Use auth state from .auth/user.json for authenticated dashboard access
- Store test data in testdata/ folder, not inline in specs

## Page Objects Required
- pages/ProjectPage.ts - Project management interactions
- pages/DashboardPage.ts - Dashboard navigation and validation
- pages/BasePage.ts - Common base methods

## Test File Structure
- File: tests/project-management.spec.ts
- Organization: 4 describe blocks (one per scenario)
- Tags: @smoke on critical tests, @project on all tests
- Before Each: Navigate to authenticated dashboard via baseURL

## Authentication & Environment
- baseURL: https://id.testsigma.com
- Credentials: TS_EMAIL and TS_PASSWORD from process.env
- Auth state: Stored in .auth/user.json
- Setup dependency: auth.setup.spec.ts runs first, creates auth state
- Test projects depend on setup project
